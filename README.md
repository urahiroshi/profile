- エンジニア組織の組織戦略の策定 (詳細: https://note.com/urahiroshi/n/n3231375a60e9, https://note.com/urahiroshi/n/n65a445a059dc)
- AIを用いた開発プロセス (AI Native Development Process) を全社に導入し、継続的に改善を実施
- 採用戦略の策定と運用、選考プロセスの刷新、グローバルエンジニア採用の立ち上げ
- エンジニアの評価基準・キャリアパスを定義し、一人EMの体制から複数EMの体制に分散
- システムリスクの評価・対策フローの策定と導入 (詳細: https://sre-next.dev/2024/schedule/#jp040)
- セキュリティリスクの評価・対策フローの策定と導入 (詳細: https://zenn.dev/dinii/articles/22cc70c70917c6)

#### 2023/02 - 2023/07 Career Break

- 個人でチャットベースのメモアプリ https://site.ducknote.app を開発 (詳細： https://urahiroshi.hatenablog.com/entry/2023/04/25/132958)

#### 2017/12 - 2023/01 株式会社メルカリ

最初は SET (Software Engineer in Test) というチームで全社的な開発・検証環境のインフラやCI/CDの運用を行う。その後、Web Platform というチームで Tech Lead, Engineering Manager を担当し、チーム運営やメルカリ Web 版のマイクロサービス化をリード。

- メルカリ Web 版のシステムアーキテクチャ設計、マイクロサービス化プロジェクトのリード、関連するマイクロサービスの立ち上げ・運用など (利用技術は React, TypeScript, Go, Kubernetes, GraphQL, gRPC, NGINX, GCP, Datadog など。詳細は https://engineering.mercari.com/blog/entry/20220830-15d4e8480e/ を参照)
- Engineering Manager として、チームの People Management や Project Management、開発プロセスや役割定義、グローバルエンジニア採用など実施 (チームについては https://engineering.mercari.com/blog/entry/20220118-introduction-of-web-platform/ を参照)
- 複数の Frontend Project 向けの CI/CD改善, モニタリングツール導入など

#### 2013/05 - 2017/11 株式会社インターネットイニシアティブ

Software Engineer として企業向け IaaS (Intrustructure as a Service) サービスの開発に従事。最初は Windows Server 環境の  IaaS サービス開発・運用が中心だったが、徐々にLinux 環境のサービス開発・運用を中心とする。

- IaaSサービスの Web Frontend の開発・運用 (CoffeeScript, Marionette.js, Ruby on Rails, MySQL, CentOS)
- Webブラウザ上で仮想サーバをコンソール画面を操作できる機能の開発・運用 (Node.js, Python, KVM, CentOS)
- IaaSサービス上で利用できる TCP ロードバランサの開発・運用 (Ruby on Rails, LVM)
- Webコントロールパネルに対するEnd-to-Endテストの実装 (CoffeeScript, Drone CI, Jenkins)
- Windows Server 用の IaaS サービスの Web Frontend, Backend 開発およびデータベース設計 (C#, SQL Server, Windows Server)

#### 2006/04 - 2013/04 セコム株式会社

総合職として入社し現場の警備業務を半年間経験した後、セキュリティシステムの開発部署を転々とし、ハードウェア設計やソフトウェア開発まで幅広く行う。
