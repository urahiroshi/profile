# Hiroshi Urayama

## 副業について

現在、副業を募集しています。副業で担当できる業務について以下に記載しておりますので、興味がある方は [SNS](#sns) よりメッセージをご送信ください。

### エンジニアのマネジメント支援

メルカリ、ダイニーでのエンジニアリングマネージャーの経験から、以下のようなマネジメント業務の支援を行うことができます。

- エンジニアおよびプロダクト開発部門の組織設計
- 人事評価プロセス構築、グレード定義
- エンジニアの育成戦略
- 採用戦略、選考フロー作成
- 組織的なブログ投稿、イベント企画などの技術広報関連業務

また、マネジメントを支援するAI対話サービス Buddy (https://buddy.learn4.jp) を開発・公開しており、マネジメント支援と並行して Buddy の導入によりマネジメントを効率化するためのサポートも実施できます (希望者のみ)。

### エンジニアリング関連業務

Webプロダクト開発やSRE関連業務に技術的な強みを持っています。個人としてWebサービスを複数リリース・運用しているほか、メルカリ時代にはメルカリWeb版を開発・運用するチームの Tech Lead を担当していました。

現在は、Learn4のプロダクト開発に集中したいため、実装・レビュー等の開発実務については基本的にご遠慮させていただきたいですが、以下のような業務や事業領域であれば担当可能です。

- 技術戦略の策定、助言
- Webサービスのアーキテクチャに関する助言
- プロダクトの品質向上、生産性向上のための仕組みづくり
- 教育・育成・マネジメントに関するプロダクトの開発 (応相談)

### その他

- Zenn の技術ブログ立ち上げ、技術イベント立ち上げ、スポンサー活動など各種技術広報活動
- エンジニアのグレード定義やValue定義を作成し、評価プロセスを体系化した上で一人EMの体制から複数EMの体制に分散

- メルカリ Web 版のシステムアーキテクチャ設計、マイクロサービス化プロジェクトのリード、関連するマイクロサービスの立ち上げ・運用など (利用技術は React, TypeScript, Go, Kubernetes, GraphQL, gRPC, NGINX, GCP, Datadog など。詳細は https://engineering.mercari.com/blog/entry/20220830-15d4e8480e/ を参照)
- Engineering Manager として、チームの People Management や Project Management、開発プロセスや役割定義、グローバルエンジニア採用など実施 (チームについては https://engineering.mercari.com/blog/entry/20220118-introduction-of-web-platform/ を参照)
- 複数の Frontend Project 向けの CI/CD改善, モニタリングツール導入など

#### 株式会社インターネットイニシアティブ (2013/05 - 2017/11)

Software Engineer として企業向け IaaS (Intrustructure as a Service) サービスの開発に従事。

- IaaSサービスの Web Frontend の開発・運用 (CoffeeScript, Marionette.js, Ruby on Rails, MySQL, CentOS)
- Webブラウザ上で仮想サーバをコンソール画面を操作できる機能の開発・運用 (Node.js, Python, KVM, CentOS)
- IaaSサービス上で利用できる TCP ロードバランサの開発・運用 (Ruby on Rails, LVM)
- Webコントロールパネルに対するEnd-to-Endテストの実装 (CoffeeScript, Drone CI, Jenkins)
- Windows Server 用の IaaS サービスの Web Frontend, Backend 開発およびデータベース設計 (C#, SQL Server, Windows Server)
